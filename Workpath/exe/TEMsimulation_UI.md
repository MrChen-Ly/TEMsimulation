# TEMsimulation5 тАФ User SOP

**Software:** TEMsimulation5.py (Dr.Probe GUI wrapper)
**Backend:** Dr.Probe suite тАФ `buildcell`, `cellmuncher`, `celslc`, `msa`, `wavimg`
**Platform:** Windows (batch `.bat` files are generated and executed automatically)

---

## Prerequisites

| Requirement | Notes |
|---|---|
| Dr.Probe binaries | `buildcell`, `cellmuncher`, `celslc`, `msa`, `wavimg` must be in system `PATH` |
| Python packages | `numpy`, `tifffile`, `matplotlib` |
| CIF file | Placed in `<workpath>\cif\` before running |
| `.prm` templates | `msa_strd.prm`, `wavimg_1.prm`, `wavimg_2.prm`, `wavimg_3.prm` in `<workpath>\prm\` |
| Folder structure | `<workpath>\cel\`, `\slc\`, `\wav\`, `\img\`, `\prm\` must exist |

---

## Working Directory Structure

```
<workpath>\
  тФЬтФАтФА cif\          тЖР place your .cif file here
  тФЬтФАтФА prm\          тЖР template .prm files (msa_strd.prm, wavimg_1/2/3.prm, MTF file)
  тФЬтФАтФА cel\          тЖР generated by buildcell / cellmuncher  (auto-cleared each run)
  тФЬтФАтФА slc\          тЖР generated by celslc
  тФЬтФАтФА wav\          тЖР generated by msa
  тФФтФАтФА img\
        тФФтФАтФА <sub_folder>\   тЖР final output images (.dat, .tif)
```

## Main GUI тАФ Parameter Reference

The GUI window is split into **two columns**.
Left column: fields 1тАУ17 | Right column: fields 18тАУ34.

---

### Column 1 тАФ Paths, Crystal & Range Setup

| # | Field label | Unit | What to enter | Example |
|---|---|---|---|---|
| 1 | **Workpath** | тАФ | Full path to the working directory. **Must end with a backslash `\`** | `D:\DrProbe\LaAlO3\` |
| 2 | **sub_workfolder** | тАФ | Name of the output sub-folder that will be created under `img\`. Each run should use a unique name to avoid overwriting results. | `run_001` |
| 3 | **cif_name** | тАФ | Filename of the CIF file (without `.cif` extension). The file must already exist at `<workpath>\cif\<cif_name>.cif`. | `LaAlO3` |
| 4 | **doc_name for save** | тАФ | Base name for generated `.cel` and `.slc` files. Usually the same as `cif_name`. | `LaAlO3` |
| 5 | **worktype** | тАФ | Simulation mode. See **Work Types** section below. | `2` |
| 6 | **a (├Е)** | ├Е | Unit cell parameter *a*. | `7.64` |
| 7 | **b (├Е)** | ├Е | Unit cell parameter *b*. | `7.64` |
| 8 | **c (├Е)** | ├Е | Unit cell parameter *c* (beam direction). | `7.64` |
| 9 | **repeat along x** | integer | Number of times to tile the unit cell along **x** (supercell). Larger values increase image size. | `3` |
| 10 | **repeat along y** | integer | Number of times to tile the unit cell along **y** (supercell). | `3` |
| 11 | **slice number** | integer | Number of slices the supercell is divided into along **z** (beam direction). More slices = better accuracy but slower. Typical: 2тАУ10 per unit cell. | `4` |
| 12 | **defocus (nm)** | nm | **Start** value of the defocus range. Positive = overfocus, negative = underfocus. | `4.5` |
| 13 | **defocus_increase (nm)** | nm | Step size between consecutive defocus values. | `1.0` |
| 14 | **defocus_end (nm)** | nm | **End** value of the defocus range. Must be тЙе field 12. | `6.5` |
| 15 | **thickness (nm)** | nm | **Start** value of the specimen thickness range. | `5.5` |
| 16 | **thickness_increase (nm)** | nm | Step size between consecutive thickness values. | `1.0` |
| 17 | **thickness_end (nm)** | nm | **End** value of the thickness range. Must be тЙе field 15. | `8.5` |

|---|---|---|---|---|

## Work Types (field 5)

| Value | Mode | Description |
|---|---|---|
| `1` | **Single image** | Generates one image at the minimum thickness (`field 15`) and minimum defocus (`field 12`). Fastest mode тАФ for quick checks. |
| `2` | **Thickness series** | Iterates over the thickness range (`fields 15тАУ17`) at a fixed defocus (`field 12`). Produces a montage map image. |
| `3` | **Defocus + thickness series** | Iterates over both defocus and thickness ranges simultaneously. Produces a full focus-thickness map. |

## Step-by-Step Workflow

1. **Prepare working directory**
   Create the folder structure shown above. Copy your `.cif` file into `cif\` and the `.prm` templates into `prm\`.

2. **Launch the GUI**
   ```
   python TEMsimulation5.py
   ```

3. **Fill in fields 1тАУ17** (left column)
   Start with **Workpath** and **cif_name**. Set the crystal parameters (a, b, c) to match your material. Choose the work type and set the defocus / thickness range.

4. **Fill in fields 18тАУ34** (right column)
   Enter aberration values for your microscope. Use manufacturer specifications or values from the microscope alignment log. If no Cs corrector is present, set Cs to its nominal value (e.g., `1.2 mm = 1.2e6 nm`).

5. **Click Submit**
   The program will:
   - Patch the `.prm` template files with your parameters
   - Write `img\runPrm.bat`
   - Execute: `buildcell` тЖТ `cellmuncher` тЖТ `celslc` тЖТ `msa` тЖТ `wavimg`
   - Export the result to a `.tif` file in `img\<sub_folder>\`
   - Display the image in a pop-up window

## Additional Features

### Tilt Loop (bottom-right of the main window)

Runs a series of simulations sweeping the **tilt azimuthal angle** at fixed magnitude.
Fill in the three small boxes before clicking **" loop"**:

| Box | Meaning | Unit |
|---|---|---|
| **beg** | Starting azimuthal angle | ┬░ |
| **inc** | Angular step | ┬░ |
| **end** | Ending azimuthal angle | ┬░ |

> The tilt magnitude comes from **field 28**. Results are saved to `img\tiltloop\`.

---

### Aberration Loop (the " loop" button)

Opens a child window for sweeping a single aberration parameter.

| Field | Meaning |
|---|---|
| **Length bg, in, ed (nm)** | Start, step, end of the aberration *magnitude* sweep |
| **Angle (deg), in (deg), end (deg)** | Start, step, end of the aberration *angle* sweep |
| **Loop type** | `1` = A1 (2-fold astig.) ┬╖ `2` = A2 (3-fold astig.) ┬╖ `3` = B2 (coma) ┬╖ `4` = defocus |

Click **" Length"** to sweep magnitude at a fixed angle, or **" Angle"** to sweep angle at a fixed magnitude. Results are saved to `img\<type>loop\`.

## Troubleshooting

| Symptom | Likely cause | Fix |
|---|---|---|
| Blank / all-zero output image | Thickness converts to slice index 0 | Increase **thickness** (field 15) or decrease **slice number** (field 11) |
| `FileNotFoundError` on `.prm` | Template `.prm` missing from `prm\` | Copy the standard `msa_strd.prm` and `wavimg_1/2/3.prm` templates |
| Output image too small / pixelated | **sample rate** too large | Decrease **field 33** (e.g., `0.005`) |
| `celslc` fails | CIF not found or wrong atom list | Check `cif_name` (field 3) and `Atoms` (field 25) |
| Very slow simulation | Supercell too large | Reduce **repeat x/y** (fields 9тАУ10) or increase **sample rate** |
| `msa` fails silently | Voltage not written correctly | Verify **Accelerating Voltage** (field 34) is a plain integer, e.g., `300` |
